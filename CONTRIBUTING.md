# Contributing

Thanks for helping grow this golden repo! To keep it consistent:

1. **Follow the existing pattern** per topic folder:
   - `README.md` — concept explanation, comparisons, when-to-use
   - code examples (`.py` / `.sql`) — runnable, commented
   - link to `resources/resources.md` for further reading (add there, not inline everywhere)

2. **Tag legacy vs modern** explicitly when adding a tool/tech, so readers understand context (e.g., "Informatica (legacy, still common in banking)").

3. **Keep code runnable** — include `pip install` lines at the top of scripts, use placeholder credentials (never real ones), and add comments explaining *why*, not just *what*.

4. **One topic per PR** — makes review easier.

5. Open an issue first for large structural changes (new top-level folders) to align before you build it out.
